# 프로젝트 완료 보고서: 독일어 학습 웹 애플리케이션

> 🇺🇸 [English Version](./Project_Completion_Report_EN)

## 1. 프로젝트 개요 및 아키텍처
**프로젝트 목표**: 현대적인 글래스모피즘(Glassmorphism) UI와 웹 음성 합성(TTS) 기능을 통합한 프리미엄 독일어 학습 웹 애플리케이션 개발.
**주요 구성 요소**:
- **루트 허브 (`index.html`)**: 전체 학습 모듈을 시각적으로 탐색할 수 있는 중앙 진입점(카드 레이아웃).
- **학습 모듈 (`Lesson/`)**: 대화형 독일어 학습 콘텐츠 뷰어.
- **어휘 모듈 (`German Vocabulary/`)**: 이미지 슬라이드쇼 및 TTS 오디오 재생이 연동된 단어장 및 갤러리.
- **비디오 학습 (`Video_Repeat.html`)**: YouTube 영상을 활용한 무한 반복 듣기 연습 페이지.
**아키텍처 및 기술 스택**: 
- HTML5, CSS3 (Vanilla CSS 적용, 글래스모피즘 디자인 패턴)
- Vanilla JavaScript (DOM 요소 제어 및 Web Speech API 활용)
- Netlify를 이용한 정적 웹 사이트 호스팅

## 2. AI 활용 전략 (Vibe Coding Environment)
프로젝트 기간 전반에 걸쳐 토큰 효율적이고 일관성 있는 AI 협업 워크플로우인 **Vibe Coding Environment**를 구축하여 활용했습니다.
- `.antigravity/current_task.md`를 통해 AI 코딩 어시스턴트에게 문맥과 목표를 명확하고 지속적으로 전달했습니다.
- "시스템 최적화(System Optimized)" 상태를 정의하고 유지함으로써, 복잡한 오류에 대한 근본 원인 분석(Root Cause Analysis) 및 빠른 패치 코드 생성을 달성했습니다.
- 단순 반복 프로세스를 AI에 위임하여 개발 기간을 단축했습니다.

## 3. 트러블슈팅 및 기술적 의사결정
- **음성 합성(TTS) 초기화 오류 및 오디오 재생 차단 문제**: 모바일 및 TV 브라우저 환경에서 보안 정책으로 인해 사용자 상호작용(User Gesture) 없이 오디오 재생이나 TTS 엔진 사용이 차단되는 문제(`speech synthesis error : synthesis - failed`)가 자주 발생했습니다.
  - **결정 및 해결**: 오디오 객체의 초기화나 TTS 실행을 명시적인 버튼(`Start` 버튼) 클릭 이벤트 안으로 전부 이관하여 브라우저의 권한 정책을 통과시켰습니다. 또한 오디오 파일의 경로 URI 인코딩 문제를 해결하여 파일 접근성을 복구했습니다.
- **Netlify 배포 시 404 에러**: 프로젝트 모듈이 모두 하위 디렉토리에만 존재할 뿐, 서버 배포의 진입점이 되는 최상위 메인 페이지가 없어 404 에러가 발생했습니다.
  - **결정 및 해결**: 루트 위치에 `index.html`(German Learning Hub)을 신규 생성하여 모든 하위 모듈(`Lesson`, `Vocabulary`, `Video_Repeat`)을 이어주는 반응형 카드 내비게이션 구조를 도입했습니다.

- **향후 과제**:

  - 모바일 해상도(태블릿, 소형 스마트폰 등) 대응을 위한 CSS 환경 최적화.
