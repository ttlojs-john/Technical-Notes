# Grafana(그라파나) 핵심 사용 가이드

Grafana는 수집된 복잡한 데이터(Prometheus 수치, Loki 로그 등)를 예쁜 그래프와 차트로 시각화해주는 대시보드 툴입니다.
처음 접속했을 때는 빈 화면이지만, **몇 번의 클릭만으로 전 세계 최고 전문가들이 만들어 둔 완성형 대시보드**를 그대로 가져와 내 서버에 적용할 수 있습니다.

## 1. 초기 로그인 및 접속
* `admin.html` 하단에 있는 **[Grafana ↗]** 버튼을 클릭하거나 `https://[도메인]/grafana/` 주소로 접속합니다.
* 초기 계정 정보
  * **아이디:** `admin`
  * **비밀번호:** `admin`
  * (로그인 직후 보안을 위해 새 비밀번호를 입력하라는 창이 뜨면, 원하는 관리자 비밀번호로 변경하세요.)

## 2. 필수 작업 1: 데이터 소스(Data Source) 연결하기
Grafana에게 "어디서 데이터를 가져와서 그릴지" 알려주는 과정입니다.
우리는 이미 도커 내부에 `Prometheus`(수치형)와 `Loki`(로그형)를 돌리고 있으므로 이 두 곳을 연결해 주어야 합니다.

1. 왼쪽 메뉴 막대(햄버거 버튼 ☰)를 클릭 -> **Connections** -> **Data sources** 로 이동합니다.
2. 우측 상단의 파란색 **[Add data source]** 버튼을 클릭합니다.
3. **[Prometheus 추가]**
   * 목록 상단에 있는 **Prometheus**를 클릭합니다.
   * `HTTP` 섹션의 `URL` 입력창에 **`http://prometheus:9090`** 이라고 입력합니다. (도커 내부 통신용 주소입니다.)
   * 화면 맨 아래로 스크롤하여 파란색 **[Save & test]** 버튼을 누릅니다. (초록색 체크 표시가 뜨면 성공!)
4. **[Loki 추가] (보안/에러 로그용)**
   * 다시 `Data sources` 메뉴 창으로 돌아와서 [Add data source]를 한 번 더 누릅니다.
   * 이번엔 목록에서 **Loki**를 찾아 클릭합니다.
   * `URL` 입력창에 **`http://loki:3100`** 이라고 입력합니다.
   * 마찬가지로 맨 아래 **[Save & test]** 버튼을 누릅니다.

---

## 3. 필수 작업 2: 남이 만든 예쁜 대시보드 가져오기 (Import)
모든 그래프를 일일이 그릴 필요가 없습니다! 전 세계 개발자들이 만들어 둔 템플릿 번호를 입력하는 것만으로 10초 만에 전문적인 상황판이 만들어집니다.

1. 왼쪽 메뉴 막대(☰) -> **Dashboards** 로 이동합니다.
2. 우측 상단의 **[New]** 드롭다운 버튼을 누른 뒤 **[Import]** 를 선택합니다.
3. 텍스트 입력창 `Find and import dashboards for...` 부분에 **`1860`** 이라는 숫자 네 자리를 적고 우측의 **[Load]** 버튼을 누릅니다.
   *(1860은 서버의 CPU, 램, 디스크 상태를 가장 완벽하게 보여주는 대표적인 템플릿 번호입니다.)*
4. Import 화면 맨 아래쪽 옵션 중에 **Prometheus** 라는 항목을 선택하는 드롭다운 박스가 있습니다.
   이것을 클릭해서 방금 위(2단계)에서 갓 만들어둔 `Prometheus` (또는 여러분이 지은 이름)를 선택해 줍니다.
5. 파란색 **[Import]** 버튼을 누릅니다!

🚀 **축하합니다! 이제 내 서버의 메모리, CPU 코어 점유율, 디스크 사용량, 네트워크 I/O가 화려한 실시간 그래프로 그려지는 상황판이 완성되었습니다!**

---

## 4. 실전 활용 팁
* **시간 범위 조절:** 대시보드 우측 상단에 🕒 시계 모양(예: `Last 1 hours`, `Last 6 hours`) 버튼을 누르면 언제부터 언제까지의 데이터를 볼지 자유롭게 조절할 수 있습니다.
* **화면 갱신 속도:** 그 바로 옆에 보면 `5s`, `1m` 등으로 자동 갱신 주기를 설정할 수 있습니다. `5s`로 두면 5초마다 우주선 관제센터처럼 화면이 실시간 반영됩니다.
* **패널(Panel) 이동 및 비율:** 각 그래프 블록의 상단을 꾹 누른 채로 드래그하면 마음대로 위치를 옮기거나 크기를 조정해서 나만의 레이아웃을 꾸밀 수 있습니다. 디자인이 맘에 들었다면 가장 우측 상단 디스크 모양(Save) 아이콘을 눌러 저장합니다.

## 5. Prometheus(프로메테우스) 직접 사용하기
Prometheus는 "숫자 데이터(Metrics)"가 저장되는 데이터베이스입니다. 그라파나가 이 데이터를 가져와서 그림을 그리는 것이죠.

* **직접 접속:** `https://[도메인]/prometheus/`
* **언제 쓰나요?**
  * 대시보드에 없는 아주 상세한 수치를 직접 쿼리해보고 싶을 때 사용합니다.
  * 예: 상단 검색창에 `node_cpu_seconds_total` 입력 후 [Execute] 클릭 -> 복잡한 숫자들이 나오면 성공입니다.
  * 보통은 그라파나가 이 작업을 대신해주므로, 일반적인 상황에서는 직접 들어갈 일이 많지 않습니다.

## 6. 로그 분석(Loki) 및 보안 점검 방법
사용자님이 질문하신 "불법 로그인 시도"나 "시스템 로그"는 **Loki**를 통해 확인할 수 있습니다.

### 🕵️‍♂️ 로그 탐색하기 (Explore)
1. 왼쪽 메뉴 막대(☰) -> **Explore** 메뉴를 클릭합니다.
2. 상단 드롭다운 박스에서 **Loki**를 선택합니다. (아까 데이터 소스에서 추가한 이름)
3. **[Label browser]** 버튼을 누르거나 쿼리창에 직접 입력하여 원하는 로그를 찾습니다.
   * **시스템 전체 로그 보기:** `{job="varlogs"}` 입력 후 [Run query]
   * **특정 파일만 보기:** `{job="varlogs", filename="/var/log/auth.log"}` (인증 관련 로그)

### 🚨 보안 로그(불법 로그인 시도) 확인법
`Explore` 창의 쿼리 입력 부분 아래에 있는 **[+ Operations]** 버튼을 활용하거나 직접 필터링 문구를 적습니다.
* **로그 필터링 예시:** `{job="varlogs", filename="/var/log/auth.log"} |= "Failed password"`
  * 이 쿼리는 "로그인 실패(Failed password)"라는 문구가 포함된 줄만 골라내줍니다.
  * 만약 특정 IP에서 수천 번의 실패 기록이 보인다면, 그것이 바로 불법 로그인 시도(Brute-force)입니다!

### 📊 대시보드에 로그 창 추가하기
이미 만들어둔 1860번 대시보드에 로그 창을 붙일 수도 있습니다.
1. 대시보드 우측 상단 **[Add]** -> **[Visualization]** 클릭
2. 데이터 소스로 **Loki** 선택
3. 쿼리에 `{job="varlogs"}` 입력
4. 우측 시각화 방식(Visualizations)에서 **Logs**를 선택하고 [Apply]를 누릅니다.
5. 이제 한 화면에서 'CPU 사용량'과 '시스템 로그'를 동시에 볼 수 있습니다!
