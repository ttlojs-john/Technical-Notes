# [Phase 3] chaeyul.uk V3.0 파이프라인 및 아키텍처 고도화 (V3.0 Pipeline Update)

**기준 기간**: 2026년 3월 15일 ~ 3월 21일

**프로젝트 개요**: Phase 3에서는 차세대 **V3.0 스마트 자막 렌더링 파이프라인**을 중점 구축하였고, 완전한 **지능형 자동 복구 모니터링 시스템**, 글로벌 다국어(i18n) 번역 엔진을 전면 통합하여 시스템 품질을 비약적으로 끌어올렸습니다.

## 🌟 1. 날짜별 핵심 고도화 내역 (Key Milestones)

### 🚀 3D 플래시카드(Flashcards) 시스템 (03.15~16)
- **비주얼 3D 플래시카드**: CSS Transform과 Flip 애니메이션 기법을 활용한 입체형 카드 학습 환경 구축.
- **간격 반복 학습 (SRS) 알고리즘**: 사용자의 숙련도를 기반으로 복습 주기를 능동적으로 조절하여 퀴즈 센터와 결합(Speed Quiz + Flashcards + Dual N-Back).
- **텔레그램 스케줄러 안정화**: `apscheduler` 객체를 FastAPI 백엔드의 비동기 시작 루프(Startup event)에 결합하여 DB 데드락 및 로딩 오류 완전 차단.

### 🎥 V3.0 유튜브 스마트 자막 백엔드 모델 (03.18)
- 기존 클라이언트 기반 호출 및 자막 로드의 한계를 극복하기 위해 서버단의 고성능 백그라운드 태스크 엔진 도입.
- 실패 영상에 대한 재시도(Retry) 및 유튜브 쿠키 권한 우회 처리를 포함한 강력한 파이프라인 큐 설계.

### 🛡️ 지능형 인프라 감시 & 보안 웹훅 복구 (03.19)
- 모니터링 스택의 **자동 복구 한계**를 극복하기 위해 알람 수신자가 텔레그램에서 '즉시 컨테이너 재부팅' 버튼을 누르면 서버가 실시간 복구하는 양방향 웹훅 고도화.
- 웹훅 실행 시 인가된 관리자 여부를 2중 검증하는 엔진 탑재로 해킹 원천 방지 및 안전한 무중단 복구 인프라 확립.
- **자동 마이그레이션(Auto-Migration) 도입**: DB 동기화 오류를 방지하기 위해 서버 재시작 시점에 구조 변경분을 스스로 판단하여 추가하는 자동 반영 엔진 가동.

### 🌍 i18n 글로벌 언어(현지화) 엔진 적용 (03.20~21)
- `i18n.js`와 `MutationObserver`를 결합하여 화면 새로고침 없이도 즉각 한글-영어 전환(`Toggle`)이 이루어지는 고속의 Incremental DOM 동기화 모듈 개발.
- 기존 속도를 제약하던 Whisper 413 Audio Error 등을 해결하는 오디오 사전 압축 알고리즘 추가로 AI 프로세싱 및 벌크 일괄 작업 큐 환경(Bulk AI Processing) 구현 완비.

## 🛠 2. 트러블슈팅 (Troubleshooting)

- **[장애] 모니터링 레이아웃 겹침**: 모니터링 시스템의 그라파나 차트가 일반 카드 컴포넌트 내부에서 z-index 및 Grid 오류를 뿜으며 겹치는 것을 발견. 다중 컬럼 체계를 스택형 싱글 프레임 구조(Vertical Stack)로 전면 포맷하여 극복.
- **[장애] 환경설정 Pydantic Null 에러**: 관리자 대시보드 저장 로직 중 입력 필드 ID 식별 누락 및 동기화 실패로 500 에러 및 데이터 증발. HTML ID 점검, DB 마이그레이션 보정 및 JSON 데이터 밸리데이션 검사를 유연하게 수정하여 해결 완료.
